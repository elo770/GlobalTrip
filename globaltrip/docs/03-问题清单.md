# GlobalTrip 项目问题清单（按优先级）

## P0 - 高优先级
1. 路线保存存在 ID 不一致风险
- 现象：前端保存时使用本地 `route.id`，后端 `POST` 会重新生成 `id`，双方可能不一致。
- 影响：后续更新 / 删除可能命中错误记录或删除失败。
- 相关文件：`src/store/trip.ts`，`server/routes/trip.js`

2. 页面层直接修改 Pinia 状态，状态边界不清
- 现象：视图层存在 `tripStore.destinations = []` 等直接赋值。
- 影响：难以统一副作用和数据同步时机。
- 相关文件：`src/views/HomeView.vue`，`src/views/MapView.vue`

## P1 - 中优先级
1. 后端仍是内存存储，重启即丢数据
- 现象：路由中使用 `let trips` / `let schedules` / `let budgetItems`。
- 影响：无法满足真实“云端存储”预期。
- 相关文件：`server/routes/trip.js`，`server/routes/budget.js`

2. POI 搜索体验与实现不一致
- 现象：防抖注释与实际时间不一致（注释写 900ms，代码为 1900ms）。
- 影响：输入到结果反馈偏慢，用户感知卡顿。
- 相关文件：`src/components/POISearch.vue`

3. POI 组件事件监听可能泄漏
- 现象：通过 `watch` 添加 `document click` 监听，但无 `onUnmounted` 兜底清理。
- 影响：多次进入页面后可能存在冗余监听。
- 相关文件：`src/components/POISearch.vue`

## P2 - 低优先级
1. Map 页面存在重复成功提示
- 现象：`handlePOISelect` 中连续弹出两次 `success`。
- 影响：交互噪音，影响体验。
- 相关文件：`src/views/MapView.vue`

2. 首页示例路线文案与实际数据不一致
- 现象：文案写“8 个城市 11 天”，示例数据更接近“5 个城市 10 天”。
- 影响：用户预期偏差。
- 相关文件：`src/views/HomeView.vue`，`src/store/trip.ts`

## 建议修复顺序
1. 统一前后端 ID 策略（先打通保存 / 更新 / 删除闭环）
2. 收敛 store 写入入口（禁止页面直接改 state）
3. 修复 POI 搜索与事件监听生命周期问题
4. 修正文案和细节交互问题
