# 运行指南

这份指南基于当前项目实际配置整理，适用于本地开发、前端部署到 Vercel，以及可选后端部署到 Render。

## 一、安装依赖

先进入项目目录：

```powershell
cd E:\product\GlobalTrip_MyWeb\globaltrip\CROSS
```

安装前端依赖：

```powershell
npm install
```

如果你需要运行本地 Express API，再安装后端依赖：

```powershell
npm install --prefix server
```

## 二、启动项目

### 只启动前端

```powershell
npm run dev
```

前端地址：

```text
http://localhost:3000
```

### 同时启动前后端

先启动前端：

```powershell
npm run dev
```

再打开第二个终端，在同一目录执行：

```powershell
npm run server
```

后端地址：

```text
http://localhost:3001
```

当前 Vite 已把 `/api` 请求代理到 `http://localhost:3001`，所以前端本地开发时不需要额外改接口地址。

## 三、环境变量

项目优先使用 Firebase，同目录下可新建 `.env`，参考 [.env.example](e:\product\GlobalTrip_MyWeb\globaltrip\CROSS\.env.example) 填入：

```env
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

可选项：

```env
VITE_FIREBASE_MEASUREMENT_ID=
```

如果不使用 Firebase，改为连接自建 API：

```env
VITE_API_BASE_URL=https://your-service.onrender.com/api
```

注意这个地址末尾要带 `/api`。

## 四、构建与预览

构建前端：

```powershell
npm run build
```

当前构建命令实际对应：

```text
vite build
```

也就是说，项目现在不会再执行 `vue-tsc && vite build`。

本地预览构建结果：

```powershell
npm run preview
```

预览地址默认是：

```text
http://localhost:4173
```

## 五、页面说明

当前页面入口如下：

1. `/home`
2. `/map`
3. `/schedule`
4. `/budget`

访问 `/` 会自动跳转到 `/home`。

## 六、部署到 Vercel

前端部署建议使用以下配置：

1. Root Directory: `CROSS`
2. Build Command: `npm run build`
3. Output Directory: `dist`

如果 Vercel 日志里仍然出现 `vue-tsc && vite build`，说明它拉到的不是最新代码，需要确认 GitHub 上 [package.json](e:\product\GlobalTrip_MyWeb\globaltrip\CROSS\package.json) 的 `build` 是否已经更新为 `vite build`。

## 七、部署到 Render

如果需要启用自建后端，可以使用当前的 [render.yaml](e:\product\GlobalTrip_MyWeb\globaltrip\CROSS\render.yaml)：

```yaml
services:
  - type: web
    name: cross-api
    runtime: node
    buildCommand: npm install --prefix server
    startCommand: node server/index.js
```

后端启动后，请确保配置好数据库和服务端环境变量。

## 八、常见检查点

1. 前端打不开时，先确认 `npm install` 是否执行成功。
2. 本地接口不通时，确认 `npm run server` 是否已启动。
3. Firebase 不生效时，检查 `.env` 或平台环境变量是否完整。
4. Vercel 仍跑旧构建命令时，先确认最新修改是否已经推送到 GitHub。
