# GlobalTrip 项目需求文档（V1 现状梳理版）

## 1. 文档目的
本文件用于梳理当前项目的真实业务需求、功能边界、角色与关键流程，作为后续迭代的统一基线。

## 2. 项目目标
- 为用户提供一个可视化旅行路线规划工具。
- 支持地图选点、路线整理、日程查看、预算记录。
- 提供可持续扩展的数据存储能力（当前已有 API 接口雏形）。

## 3. 用户角色100010                           00000                                                    
- 普通用户（当前唯一角色）
- 使用场景：
- 创建并编辑个人旅行路线
- 查看路线对应日程
- 记录预算明细与统计

## 4. 功能范围（当前已实现）

### 4.1 首页
- 查看已保存路线列表
- 创建新路线
- 加载示例路线（改革开放主题）
- 删除已保存路线

### 4.2 地图规划页
- 地图展示（OpenLayers）
- POI 搜索并添加点
- 地图点击添加点（带弹窗表单）
- 编辑点（名称、描述、日期、图片）
- 点排序上下移动
- 删除点
- 保存路线
- 地图工具：平移、放大、缩小、适应范围

### 4.3 日程页
- 按天展示日程时间线
- 从当前路线自动生成日程
- 一键加载示例路线

### 4.4 预算页
- 新增/删除预算项
- 预算明细表
- 按类型/按天汇总
- 饼图可视化

## 5. 数据与状态（当前实现）
- 前端核心状态在 Pinia（`trip` store）：
- `destinations`
- `currentRoute`
- `schedule`
- `budgetItems`
- `savedRoutes`
- 本地持久化：`savedRoutes` 使用 `localStorage`
- 后端同步（已接入）：
- 路线：`/api/trips`
- 预算：`/api/budget`

## 6. 接口需求（当前定义）
- `GET /api/trips`：获取路线列表
- `POST /api/trips`：创建路线
- `PUT /api/trips/:id`：更新路线
- `DELETE /api/trips/:id`：删除路线
- `GET /api/budget`：获取预算项（可按 `tripId` 过滤）
- `POST /api/budget`：新增预算项
- `PUT /api/budget/:id`：更新预算项
- `DELETE /api/budget/:id`：删除预算项
- `GET /api/health`：服务健康检查

## 7. 非功能需求（建议）
- 可用性：
- 关键操作（保存、删除）需反馈明确成功/失败状态
- 性能：
- 地图交互保持流畅（常规设备下点击与拖拽无明显卡顿）
- 可靠性：
- 后端不可用时不应导致页面崩溃
- 可维护性：
- 页面和 store 的职责边界清晰
- 安全性：
- 未来引入鉴权、输入校验和基础限流

## 8. 暂不纳入范围（当前阶段）
- 多用户/登录体系
- 分享协作
- 真实交通路径规划引擎
- 真实订单或支付能力

## 9. 验收标准（V1）
- 用户可在地图页创建并保存路线，并在首页可再次打开。
- 用户可管理预算项并看到统计图表。
- 前端与后端 API 连通时，刷新后数据可从服务端拉取。
- 后端异常时，前端功能保持可用并提示错误（不白屏）。
